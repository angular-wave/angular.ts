AngularTS Runtime Concepts

These demos treat the DOM as one runtime among many. AngularTS owns the app model and DOM projection; each external runtime owns its native scheduler, rendering objects, and lifecycle.

Run make serve, then open any concept below.

Browser Drawing Runtimes

  • Canvas 2D canvas state projected from an app model.
  • WebGL Low-level GPU drawing controlled by AngularTS state.
  • WebGPU Capability-gated GPU clear color driven by a model.
  • HTML-in-Canvas Native layoutsubtree rendering with AngularTS controls.
  • AudioGraph Web Audio graph controlled by DOM and app model state.

3D And Rendering Runtimes

  • Three.js Model-driven cube material and rotation speed.
  • BabylonJS Shark scene with DOM controls bound to sharkModel.
  • PlayCanvas 3D entity state projected from cubeModel.

Game Runtimes

  • Cockpit Breach HTML-in-Canvas cockpit game and hackathon demonstrator.
  • Glyph Rush AngularTS model-driven page defense game.
  • Phaser Canvas game runtime sharing playerModel with DOM.
  • PixiJS 2D renderer adapter with model-driven sprite state.
  • Excalibur 2D game engine actor driven by playerModel.
  • Unity FPS Unity WebGL FPS scene bridged into AngularTS models.

Data Visualization Runtimes

  • ECharts Chart runtime reads model data and writes selection back.
  • D3 Low-level SVG chart projection with model-backed selection.

Data Grid Runtimes

  • Virtual Grid Virtualized rows project selection and viewport state into gridModel.

CAD, Design, And Canvas Object Runtimes

  • Konva Draggable canvas object writes position back into shapeModel.

Editor Runtimes

  • CodeMirror Editor text and cursor projected into documentModel.

Terminal Runtimes

  • xterm.js Terminal input and connection state projected into terminalModel.

Document And PDF Runtimes

  • PDF.js PDF page and zoom state coordinated through documentViewerModel.

Map Runtimes

  • MapLibre GL Map camera and marker state shared through mapModel.

Worker, WASM, And Data Runtimes

  • Web Worker Compute Worker results synchronized into computeModel through $sync.
  • Shared WASM Module Worker One compiled module creates independent main-thread and worker instances.
  • SQLite WASM SQLite query runtime projects rows into databaseModel.

Local-First And Collaboration Runtimes

  • Yjs CRDT document updates synchronized into documentModel.
